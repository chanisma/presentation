🧠 알고리즘이란?

알고리즘 — 이름의 유래

"algorithm"은 어디서 왔을까요?

무함마드 이븐 무사 알 콰리즈미 (780~850년경)

📖

알 콰리즈미 (Al-Khwārizmī)

9세기 페르시아의 수학자이자 천문학자. 그의 이름 "알 콰리즈미"가 라틴어로 전해지면서 "알고리즘"이라는 단어가 만들어졌습니다.

📐

수학적 업적

대수학(Algebra)의 기초 확립, 0과 10진법 도입, 문제를 해결하는 절차적 접근법 제시

🌐

과학적 업적

천문학 및 삼각법을 활용한 계산법 발전, 지리학 연구와 경도·위도 개념 보완

핵심 정의
알고리즘 = 문제를 해결하는 절차적 접근법
1,200년 전 수학자의 아이디어가 현대 컴퓨터 과학의 기반이 되었습니다

명확한 순서

각 단계는 누가 봐도 이해할 수 있을 만큼 명확합니다. 모호한 지시는 알고리즘이 아닙니다.

⏱️

반드시 끝남

무한히 실행되지 않습니다. 언젠가는 결과를 내고 종료합니다.

🔧 알고리즘의 3가지 구조

모든 알고리즘은 3가지 구조로 이루어집니다

아무리 복잡한 프로그램도 이 세 가지의 조합입니다

📋
① 순차 구조
시작부터 종료까지 순서대로 명령을 실행하는 구조
예: 라면 끓이기
물 끓이기 → 면 넣기 → 수프 넣기 → 3분 기다리기 → 완성
🤔 어떤 문제에 순서가 필요할까요?
🔀
② 선택 구조
조건에 따라 실행하는 명령이 달라지는 구조
예: 음악 추천
기분이 좋으면 → 신나는 노래
기분이 우울하면 → 잔잔한 노래
🤔 어떤 조건이 필요한지 어떻게 정할까요?
🔁
③ 반복 구조
조건이 만족될 때까지 특정 명령을 반복 실행하는 구조
예: 인터넷 검색
원하는 정보가 나올 때까지
스크롤하며 결과 확인 반복
🤔 반복이 멈추는 조건은 무엇인가요?
💡 핵심: 이 세 구조를 조합하면 세상의 어떤 문제도 알고리즘으로 표현할 수 있습니다. 하노이의 탑 게임에서도 이 세 구조를 모두 찾을 수 있어요!
✅ 알고리즘으로 해결할 수 있는 문제

알고리즘으로 모든 문제를 풀 수 있을까요?

모든 문제가 알고리즘으로 해결되는 것은 아닙니다 — 4가지 조건이 필요합니다

🗼 하노이의 탑 활동을 다시 떠올려 보세요. 그 게임이 알고리즘으로 해결될 수 있었던 이유가 무엇일까요?
🎯
① 명확성 (Definiteness)
문제와 각 단계가 모호하지 않고 명확하게 정의되어야 합니다.
↔️
② 입력과 출력 (I/O)
입력 내용과 출력 내용이 구체적으로 정의되어야 합니다.
⏳
③ 유한성 (Finiteness)
유한한 단계를 거쳐 반드시 끝나야 합니다. 무한 루프가 없어야 합니다.
⚙️
④ 실현 가능 (Feasibility)
각 단계는 실제로 실행 가능한 방법이어야 합니다. 불가능한 단계가 없어야 합니다.
이 4가지가 모두 충족되어야 알고리즘으로 해결 가능한 문제가 됩니다
🎯 조건 ① — 명확성

① 명확성  문제가 분명하게 정의되어야 합니다

"누가 봐도 같은 방법으로 풀 수 있을 만큼" 명확해야 합니다

✅ 명확한 문제
✓
"학교에서 집까지 최단 경로(최소 이동 거리)를 찾아라"
👍 "최단 경로"는 수치로 측정 가능 — 기준이 명확합니다.
❌ 모호한 문제
✗
"학교 가는 길을 편하게 가는 방법을 찾아라"
🤔 "편하다"의 기준이 사람마다 다릅니다. 컴퓨터는 모호한 기준을 처리할 수 없어요.
🗼

하노이의 탑 — 명확성 체크 ✅

규칙이 명확합니다:
• 큰 원반은 작은 원반 위에 올라갈 수 없다
• 한 번에 하나의 원반만 옮길 수 있다
목표가 명확합니다:
• A 기둥의 원반을 C 기둥으로 옮긴다
• 최소 이동 횟수로 달성한다
💡 핵심: 알고리즘은 컴퓨터가 실행합니다. 컴퓨터는 "대충", "적당히", "편하게" 같은 모호한 표현을 이해하지 못합니다. 수치와 규칙으로 정확히 정의해야 합니다.
↔️ 조건 ② — 입력과 출력

② 입력과 출력  무엇을 넣고 무엇이 나오는지 정확히

알고리즘은 "블랙박스"처럼 — 입력을 받아 처리하고 출력을 냅니다

🗺️

예시 1 — 길 찾기

📥 입력: 출발지, 목적지, 도로 정보
📤 출력: 최단 경로, 예상 소요 시간
🎵

예시 2 — 음악 추천

📥 입력: 학생의 청취 기록, 좋아요/싫어요 이력
📤 출력: 다음 추천 곡 목록 (상위 10개)
❌ 입출력 불분명
✗
"완벽한 친구 추천 시스템을 만들어라"
"완벽한 친구"의 기준이 없음 — 출력이 정의되지 않음
🗼

하노이의 탑 — 입출력 ✅

📥 입력: 기둥 A의 원반 개수(n개)
📤 출력: 원반이 모두 C 기둥에 쌓인 상태
→ 원반 3개이면 최소 7번의 이동 기록
⏳⚙️ 조건 ③④ — 유한성 & 실현 가능

③ 유한성 & ④ 실현 가능

언젠가는 끝나야 하고, 각 단계는 실제로 할 수 있어야 합니다

③ 유한성 — 반드시 끝나야 합니다
✓
학생들을 키 순서대로 정렬
→ 학생 수(n)가 정해져 있으므로 최대 n²번 비교 후 반드시 종료
✗
"완벽한" 답이 나올 때까지 계속 계산
→ 완벽의 기준이 없으면 영원히 끝나지 않을 수 있음
🗼

하노이의 탑 — 유한성 ✅

원반 n개일 때, 정확히 2ⁿ−1번 이동 후 완료.
원반 3개 → 7번, 10개 → 1023번. 항상 끝납니다.

④ 실현 가능 — 실제로 할 수 있는 단계
✓
버블 정렬로 학생들을 키 순으로 정렬
→ 각 단계(두 학생 키 비교 후 교환)는 누구나 할 수 있음
✗
"학생들끼리 대화해서 가장 인기 있는 사람을 찾음"
→ 어떻게 대화를 수치화하고 처리할지 정의되지 않음
🗼

하노이의 탑 — 실현 가능 ✅

각 단계: "원반 하나를 한 기둥에서 다른 기둥으로 옮기기"
누구든 실제로 할 수 있는 명확한 행동입니다.

🎮

CodeCombat이란?

RPG 게임 형식으로 실제 프로그래밍 언어(Python/JavaScript)를 배우는 플랫폼입니다. 코드를 작성하면 게임 캐릭터가 그대로 움직입니다!

📝

오늘 배울 코드

hero.moveRight()
hero.moveDown()
hero.moveLeft()
hero.moveUp()
🔗 수업 코드로 접속하기
📋 접속 코드: StepJumpTooth
위 링크를 클릭하거나 codecombat.com에서 학생으로 가입하세요

CodeCombat 학생 계정 만들기

  • 3
