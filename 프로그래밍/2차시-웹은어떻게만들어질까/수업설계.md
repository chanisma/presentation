# 프로그래밍 2차시 - 웹은 어떻게 만들어질까?

## 기본 정보

| 항목 | 내용 |
|------|------|
| 수업명 | 프로그래밍 |
| 차시 | 2차시 |
| 제목 | 웹은 어떻게 만들어질까? |
| 대상 | 고등학교 여학생 (웹 프로그래밍 입문) |
| 수업 위치 | 1차시(오리엔테이션) → **2차시** → 이후 IDE 설치/HTML 코딩 |

## 학습 목표

1. 인터넷과 웹의 차이를 안다
2. 웹사이트가 화면에 보이는 과정(클라이언트-서버)을 이해한다
3. 웹사이트의 구조(헤더/내비게이션/메인/푸터)를 분석할 수 있다

## 1차시와의 차별점

| 1차시에서 다룬 것 (중복 피함) | 2차시에서 새로 다룰 것 |
|------|------|
| 프로그래밍을 배우는 이유 | 인터넷 vs 웹의 차이 |
| HTML=뼈대, CSS=디자인, JS=동작 (한 줄 소개) | 집 짓기 비유로 더 구체적 이해 |
| 3트랙 소개 (데이터/웹/바이브코딩) | 클라이언트-서버 작동 원리 |
| 도구/일정/평가 안내 | 웹사이트 구조 분석 활동 |

## 수업 흐름

### Intro (도입)

**슬라이드 1: 표지**
- 프로그래밍 2차시 - 웹은 어떻게 만들어질까?
- 부제: "우리가 매일 사용하는 웹사이트의 비밀"

**슬라이드 2: 지난 시간 복습**
- 1차시 3가지 트랙 리마인드: 데이터 연구 / 웹 언어 / 바이브 코딩
- "오늘부터 본격적으로 '웹 언어' 트랙을 시작합니다!"

**슬라이드 3: 오늘의 질문**
- "여러분이 매일 보는 웹사이트... 어떻게 만들어졌을까?"
- 인스타그램, 유튜브, 네이버 등 학생 친숙 서비스 예시
- 오늘의 학습 목표 제시

### Explore (탐구)

**슬라이드 4: 인터넷 vs 웹**
- 인터넷 = 전 세계 컴퓨터를 연결하는 거대한 네트워크 (고속도로)
- 웹(WWW) = 인터넷 위에서 동작하는 서비스 중 하나 (버스)
- 인터넷 위에는 웹 외에도 이메일, 메신저, 게임 등 다양한 서비스

**슬라이드 5: 웹사이트가 보이는 과정 ①**
- 카페 주문 비유로 클라이언트-서버 개념 소개
  - 나(클라이언트/브라우저) → 주문(요청/URL 입력) → 바리스타(서버) → 커피 전달(응답/웹페이지)
- 핵심 메시지: 웹사이트는 서버에 저장된 파일을 브라우저가 받아서 보여주는 것

**슬라이드 6: 웹사이트가 보이는 과정 ②**
- 단계별 시각화: URL 입력 → 서버에 요청 → 서버가 HTML/CSS/JS 파일 전달 → 브라우저가 화면에 표시
- 각 단계를 순서도/플로우차트 형태로 정리

**슬라이드 7: 웹페이지의 3가지 언어 ①**
- 1차시 내용을 집 짓기 비유로 심화
  - HTML = 설계도 & 골조 (방, 벽, 문, 창문의 위치)
  - CSS = 인테리어 (벽지 색, 가구 배치, 조명)
  - JavaScript = 전자기기 (스위치 누르면 불 켜짐, 리모컨 동작)

**슬라이드 8: 웹페이지의 3가지 언어 ②**
- 간단한 코드 맛보기 시각 자료
  - HTML: `<h1>안녕하세요!</h1>`
  - CSS: `color: blue; font-size: 24px;`
  - JS: `alert('클릭!')`
- "이런 코드를 앞으로 직접 작성하게 됩니다!"

### Apply (적용)

**슬라이드 9: 웹사이트 구조 분석**
- 미니 웹사이트 목업(학교 사이트)으로 4가지 영역 시각적 체험
  - 헤더(Header): 로고, 메뉴 - 사이트 상단 (빨간색)
  - 내비게이션(Nav): 메뉴, 링크 - 페이지 이동 (주황색)
  - 메인 콘텐츠(Main): 본문 내용 - 핵심 정보 (파란색)
  - 푸터(Footer): 연락처, 저작권 - 사이트 하단 (회색)
- 각 영역의 역할 설명 (사이트의 얼굴, 길 안내 표지판, 심장, 부록)

**슬라이드 10: 소스 코드 보는 방법**
- 방법 1: Ctrl+U (페이지 소스 보기)
- 방법 2: F12 또는 Ctrl+Shift+I (개발자 도구)
- 마우스 오른쪽 클릭 → "검사" (특정 요소 바로 이동)
- 개발자 도구 미니 목업: 코드와 화면이 연결되는 모습 시각화
- header/nav/main/footer 태그가 4가지 영역과 대응됨을 발견

**슬라이드 11: 활동 - 웹사이트 구조 찾기**
- 활동 안내:
  1. 좋아하는 웹사이트 1개를 선택
  2. 눈으로 헤더/내비/메인/푸터 영역 찾기
  3. F12로 개발자 도구 열어 코드에서 태그 확인
  4. 활동지에 각 영역의 내용과 태그 이름 기록
  5. 모둠원과 분석 결과 비교
- 찾아볼 태그: header, nav, main, footer, div
