개념 정의
What is Ontology?

온톨로지란 무엇인가?

🏛️ 철학에서의 온톨로지

존재론(存在論) — "존재하는 것은 무엇인가?"를 탐구하는 철학의 한 분야

  • 아리스토텔레스의 범주론에서 시작
  • 존재의 본질과 구조를 분류
  • 세계를 이해하는 개념적 틀 제공

💻 정보과학에서의 온톨로지

"공유된 개념화의 형식적이고 명시적인 명세" — Tom Gruber (1993)

  • 특정 분야의 지식을 체계적으로 표현
  • 개념, 속성, 관계를 정의
  • 컴퓨터가 이해할 수 있는 형식
구성요소
Core Components

온톨로지의 4가지 핵심 구성요소

🏷️

클래스 (Class)

같은 특성을 가진 개체들의 집합. 예: "동물", "포유류", "개"

🔗

관계 (Relation)

클래스 간의 연결. 예: "개 → 하위분류 → 포유류"

📋

속성 (Property)

클래스의 특징을 기술. 예: "이름", "나이", "서식지"

📌

인스턴스 (Instance)

클래스에 속하는 구체적 개체. 예: "내 반려견 초코"

구조 다이어그램
Structure Diagram

온톨로지의 계층 구조

🖼️ 이미지를 불러올 수 없습니다

온톨로지의 클래스-서브클래스 계층 구조 (출처: Stanford/Noy & McGuinness)

📊 구조 읽는 법

상위 클래스(Thing) → 하위 클래스(생물/무생물) → 더 구체적 클래스(동물/식물) → 인스턴스(초코)로 내려갑니다. 이것이 is-a 관계입니다.

RDF 트리플
RDF Triple

온톨로지의 표현 방법: RDF

RDF(Resource Description Framework)는 온톨로지를 표현하는 기본 단위입니다.

주어 (Subject) → 서술어 (Predicate) → 목적어 (Object)
세종대왕 —발명→ 한글
세종대왕 —직업→ 왕
한글 —유형→ 문자체계
코드로 보면

<세종대왕> <발명> <한글> .
<세종대왕> <유형> <왕> .
<한글> <유형> <문자체계> .

구축 과정
Building Process

온톨로지는 어떻게 만드는가?

  1. 1
    도메인 범위 결정
    어떤 분야의 지식을 표현할 것인가? (예: 의료, 교육, 음식)
  2. 2
    핵심 개념(클래스) 나열
    해당 분야의 주요 개념들을 브레인스토밍으로 수집
  3. 3
    계층 구조 설계
    상위/하위 관계(is-a)로 클래스를 조직화
  4. 4
    속성과 관계 정의
    각 클래스의 속성(has-a)과 클래스 간 관계를 명세
  5. 5
    인스턴스 생성 및 검증
    실제 데이터를 입력하고 추론 엔진으로 일관성 확인
