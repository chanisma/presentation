# 2026 정보 활동지 제출 현황 CSV 생성 - Design

## 목적
1학년 학생 227명(Supabase `resource.meta.grade=1`)이 6개 시트에 등록된 과제를 제출했는지 확인하여 CSV로 저장한다.

## 입력
- **Excel 파일**: `C:/Users/User/Downloads/2026 정보 활동지.xlsx`
  | Idx | Sheet name | Identifier columns | Note |
  |-----|------------|-------------------|------|
  | 0 | 인공지능 시대 | `학번과 이름을 적어주세요` (결합) | 예: "10224 최하진" |
  | 1 | 대형언어모델 | `이름과 학번` (결합) | 예: "10111 박주연" |
  | 2 | 자기평가문활동지 | `이름`, `반/번호` | 예: "3반 13번" |
  | 3 | 에이전틱AI활동지 | `이름`, `반/번호` | 예: "1반 1번" |
  | 4 | 딥러닝기초활동지 | `이름`, `반/번호` | 현재 데이터 없음 |
  | 5 | 이미지분류프로젝트활동지 | `이름`, `반/번호` | 예: "1반 22번" |

- **Supabase**: 프로젝트 `oumtnqamlpwljkrgbiqg`, `resource` 테이블
  - 쿼리: `SELECT name, meta FROM resource WHERE meta->>'grade' = '1'`
  - 필드: `name` (string), `meta->>'class'`, `meta->>'number'`

## 학생 매칭 전략

### 반/번호 정규화
학생마다 다른 표기 양식을 통일:
- `"3반 13번"` → `(class=3, number=13)`
- `"3-13"` / `"3 13"` / `"3반13"` / `"1반 22번"` → `(class, number)`
- `"10313"` (학년+반+번호) → 5자리면 앞 1자리 학년 제거 후 `(3, 13)`
- `"313"` (반+번호 2자리) → `(3, 13)` (class 1자리, number 나머지)
- `"119"` / `"122"` → class 1자리, 나머지는 number (2자리면 `(1, 19)`, `(1, 22)`)

### 결합 식별자 파싱
`"10224 최하진"` 형태:
- 공백/특수문자로 split → 숫자 토큰과 한글 토큰 분리
- 숫자 토큰 = 학번 (5자리 표준: 학년1 + 반1~2 + 번호2)
- 한글 토큰 = 이름

### 매칭 키
`(name, class, number)` 3개 모두 일치해야 매칭 성공.
- Supabase 학생의 `(name, class::int, number::int)` vs 제출 데이터의 `(parsed_name, parsed_class, parsed_number)`

### 매칭 실패 처리
- 매칭 안 된 제출은 stderr에 출력 (row reference, 시트명, 원본 식별자)
- 실패율도 stderr에 집계

## 제출 집계
각 학생 × 각 시트에 대해:
```json
{
  "count": 2,
  "times": ["2026-03-13T09:21:51", "2026-03-14T10:00:00"]
}
```
제출 없음: `{"count": 0, "times": []}`

타임스탬프 파싱:
- datetime 객체면 `.isoformat()`
- 문자열이면 알려진 포맷 시도 (`"2026. 4. 6. AM 11:32:18"` 등)
- 파싱 실패 시 원본 문자열 유지

## 동일 결과물 탐지

### 해시 계산
각 시트 내에서 각 제출 행에 대해:
1. 답변 컬럼만 추출 (타임스탬프, 이름/학번, 반/번호 제외)
2. 각 셀 값을 문자열화 + 공백 정규화(`re.sub(r'\s+', ' ', s).strip()`) + null → `""`
3. 구분자 `\x1f`로 join
4. SHA256 해시

### 중복 그룹 탐지
- 시트 내에서 동일 해시를 가진 서로 다른 학생(매칭 키 기준)이 2명 이상 → 중복 그룹
- 같은 학생이 같은 해시로 여러 번 제출한 것은 중복 아님
- 학생의 여러 제출 중 하나라도 다른 학생과 해시 일치하면 중복 처리

### duplicate_groups 형식
```json
{
  "대형언어모델": ["10111 박주연", "10215 김철수"],
  "에이전틱AI": ["10101 강다원", "10102 홍길동"]
}
```
- key: 시트명
- value: 자기 자신을 제외한 동일 결과물 제출자 리스트 (학번 + 이름)
- 중복 없으면 `{}`

## 출력 CSV

### 파일 경로
`C:/repository/presentation/활동지_제출현황.csv`
- 인코딩: UTF-8 with BOM (Excel 한글 호환)

### 컬럼
```
grade, class, number, name,
인공지능시대, 대형언어모델, 자기평가문, 에이전틱AI, 딥러닝기초, 이미지분류,
duplicate_groups
```

### 값 형식
- `grade`, `class`, `number`: 정수
- `name`: 문자열
- 시트 컬럼: JSON 문자열 `{"count": N, "times": [...]}`
- `duplicate_groups`: JSON 문자열

### 정렬
`class ASC, number ASC`

## 실행 방법
Python 스크립트 `scripts/build_submission_report.py`:
1. Supabase MCP로 학생 227명 조회
2. Excel 파일 openpyxl로 로드
3. 시트별 파싱 + 매칭
4. 중복 해시 집계
5. CSV 출력
6. stderr에 통계 (총 제출 수, 매칭 성공/실패, 중복 수)

의존성: `openpyxl` (이미 환경에 있을 가능성 높음, 없으면 pip install)

Supabase 조회는 MCP 도구로 Claude가 직접 수행하고, 결과를 JSON 파일로 저장한 후 Python 스크립트가 읽도록 한다. (스크립트가 직접 Supabase에 접속하지 않아 자격증명 불필요)

## 검증
1. 출력 CSV 행 수 = 227
2. 시트별 매칭된 제출 수 + 미매칭 수 = 시트 원본 제출 수
3. 중복 그룹 샘플 수동 확인
