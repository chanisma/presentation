MNIST 데이터셋이란?

딥러닝의 "Hello World" - 손글씨 숫자 이미지 데이터셋

핵심 정보

  • 70,000장의 손글씨 숫자 이미지 (학습 60,000 + 테스트 10,000)
  • 28 x 28 픽셀 흑백(grayscale) 이미지
  • 0 ~ 9까지 10가지 분류
  • 1998년 Yann LeCun이 제작
각 픽셀은 0(흰색)~255(검정) 사이의 숫자값으로 표현됩니다

숫자 "5"의 간략한 픽셀 표현

밝을수록 높은 픽셀값

컴퓨터가 이미지를 이해하는 방법을 알아봅시다

1

사람이 쓴 숫자 이미지

✍️

종이에 쓴 숫자를 스캔하거나 디지털로 입력

2

28 x 28 격자로 나누기

📏

784개의 작은 픽셀로 분할

3

밝기를 숫자로 변환

🔢

각 픽셀의 밝기를 0~255 값으로 저장

💡 컴퓨터는 이미지를 '숫자의 배열'로 인식합니다. 28x28 = 784개의 숫자가 하나의 손글씨 이미지를 표현합니다.
이 변환 과정을 '전처리(Preprocessing)'라고 합니다. 모든 이미지를 같은 크기, 같은 형식으로 맞추는 것이 중요합니다.

입력 이미지가 숫자로 분류되기까지의 전체 과정

입력 28x28
→

→
ReLU 활성화
→
풀링 크기 축소
→

→

→

→
출력 0~9
지난 시간에 배운 CNN 구조를 MNIST 숫자 인식에 적용합니다. 각 단계가 어떤 역할을 하는지 다음 슬라이드에서 자세히 살펴봅시다.
CNN 각 단계의 역할

CNN 각 단계의 역할

각 단계가 숫자 인식에서 어떤 일을 하는지 알아봅시다

🔍

컨볼루션 (Convolution)

필터가 이미지를 스캔하며 가장자리, 곡선 등의 특징을 추출합니다

⚡

ReLU (활성화 함수)

음수값을 0으로 만들어 중요한 특징만 남기고 불필요한 신호를 제거합니다

📐

풀링 (Pooling)

특징 맵의 크기를 줄여 핵심 정보만 유지하고 계산량을 감소시킵니다

📊

Flatten (평탄화)

2D 특징 맵을 1D 벡터로 변환하여 분류기에 입력할 수 있도록 합니다

🧠

완전연결층 (Dense)

추출된 모든 특징을 종합하여 어떤 숫자인지 판단합니다

🎯

Softmax (출력)

각 숫자(0~9)일 확률을 0~1 사이 값으로 계산합니다

CNN 개념 코드 (Keras) 역할
입력 이미지 input_shape=(28,28,1) 28x28 흑백 이미지 입력
컨볼루션 Conv2D(32, (5,5)) 5x5 필터 32개로 특징 추출
활성화 함수 activation='relu' 비선형성 추가, 음수 제거
풀링 MaxPooling2D((2,2)) 크기 축소 (28x28 → 14x14)
평탄화 Flatten() 2D → 1D 벡터 변환
완전연결 Dense(1024) 특징 조합하여 판단
출력 Dense(10, softmax) 0~9 각 확률 출력
