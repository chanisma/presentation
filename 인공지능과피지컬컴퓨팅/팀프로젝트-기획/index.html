◆ 사용 가능 엑추에이터 카탈로그

Tier 1 — 수업에서 이미 사용한 기본 부품

LED (단색)

출력: 빛 (ON/OFF, PWM 밝기 조절)
함수: digitalWrite() / analogWrite()
핀: 디지털 핀 (PWM 핀 권장)
활용: 상태 표시등, 경고 신호, 밝기 단계 표현
주의: 220Ω 저항 필수

★☆☆ 쉬움

RGB LED

출력: 빛 (R·G·B 색상 조합)
함수: analogWrite() × 3핀
핀: PWM 핀 3개 사용
활용: 감정→색상 매핑, 분류 결과별 색상 표시
주의: 각 핀에 220Ω 저항 필요

★☆☆ 쉬움

피에조 부저

출력: 소리 (주파수로 음높이 제어)
함수: tone(핀, 주파수) / noTone()
핀: 디지털 핀 아무 곳이나
활용: 경고음, 도레미 멜로디, 알림 신호
주의: +/- 극성 확인

★☆☆ 쉬움

SG90 서보모터

출력: 회전 (0°~180° 각도 제어)
함수: Servo.attach() / Servo.write()
핀: PWM 핀 (D9, D10 등)
활용: 문 개폐, 방향 지시, 팔 동작
주의: 5V 전원 필수, 3.3V 부족

★★☆ 보통

◆ 엑추에이터 카탈로그 (계속)

Tier 2 — 추가 학습이 필요하지만 도전할 만한 부품

진동 모터

출력: 진동 (ON/OFF)
함수: digitalWrite()
핀: 디지털 핀 + 트랜지스터 권장
활용: 촉각 알림, 무음 경고, 햅틱 피드백
주의: 전류 소모 주의, 트랜지스터로 구동 권장

★☆☆ 쉬움

5V 릴레이 모듈

출력: 스위칭 (외부 기기 ON/OFF)
함수: digitalWrite()
핀: 디지털 핀 1개
활용: USB 선풍기·조명 켜기/끄기
주의: 5V 기기만 제어 (절대 220V 연결 금지)

★★☆ 보통

I2C LCD (16×2)

출력: 텍스트 표시 (2줄 × 16자)
함수: LiquidCrystal_I2C 라이브러리
핀: SDA(A4), SCL(A5) — 2핀만 사용
활용: 인식 결과 문자 출력, 상태 메시지
주의: I2C 주소 확인 필요 (보통 0x27)

★★☆ 보통

DC 모터 + L298N

출력: 연속 회전 (양방향, 속도 제어)
함수: analogWrite() + digitalWrite()
핀: PWM 1개 + 디지털 2개
활용: 바퀴 구동, 팬 회전, 컨베이어
주의: 외부 전원 필요할 수 있음, 모터 드라이버 필수

  1. 01
    팀 정보

    팀명, 팀원 이름, 역할 분담 (AI학습 / 회로 / 코딩 / 발표)

  2. 02
    문제 정의

    한 문장: "○○ 상황에서 ○○ 문제를 ○○ 방식으로 해결한다"

  3. 03
    해결 방법

    웹캠으로 무엇을 인식하고, 어떤 엑추에이터가 어떻게 반응하는지 (입력→분류→출력 흐름)

  4. 04
    클래스 설계

    TM 모델의 클래스명과 각 클래스별 엑추에이터 동작 매핑 (표 형태)

  5. 05
    필요 부품

    사용할 엑추에이터 + 기타 부품 목록 (화이트리스트 내 확인)

  6. 06
    차시별 일정

    차시별 할 일 (1차시: 기획, 2차시: 제작+테스트, 3차시: 완성+발표)

  7. 07
    기대 효과

    이 프로젝트가 해결하는 가치 한 줄 + 확장 가능성

⊕ 자주 묻는 질문

  • ①
    화이트리스트에 없는 부품 써도 되나요?

    교사에게 사전 승인을 받으면 가능합니다. 단, 5V 범위 내 안전한 부품이어야 합니다.

  • ②
    TM 모델은 새로 만들어야 하나요?

    지난 시간에 만든 모델을 재사용해도 되고, 새 모델을 학습시켜도 됩니다.

  • ③
    코딩을 잘 못하는데 괜찮을까요?

    AI에게 코드 생성을 요청하는 방법을 이미 배웠습니다. 기존 코드 구조 + 구체적 요구사항을 프롬프트로 제공하세요.

  • ④
    아이디어가 안 떠올라요

    일상에서 불편한 점 3가지를 먼저 적어보세요. 슬라이드 8의 예시도 참고하세요.

  • ⑤
    발표는 어떻게 하나요?

    프로토타입 시연 + 기획서 기반 2분 발표입니다. 모든 팀원이 한마디씩 참여하세요.
